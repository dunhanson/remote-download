## BullMQ 介绍

**BullMQ** 是基于 Node.js 的任务队列库，构建在 Redis 之上，是 Bull 的现代继任者。

### 核心特性

- **异步任务处理**：将耗时的操作（如下载、邮件发送）放入队列异步处理
- **任务持久化**：任务存储在 Redis 中，服务重启不会丢失
- **重试机制**：支持配置重试次数和退避策略（指数退避、固定延迟）
- **并发控制**：可限制同时运行的任务数量
- **优先级队列**：支持任务优先级排序
- **延迟任务**：支持延迟执行任务
- **分布式**：多个进程/服务器可以共享同一个队列

### 基础概念

| 概念 | 说明 |
|------|------|
| **Queue** | 队列容器，管理所有任务 |
| **Job** | 具体任务单位，包含数据和配置 |
| **Worker** | 消费者，从队列获取并执行任务 |
| **Producer** | 生产者，向队列添加任务 |

### 基础用法

```typescript
import { Queue, Worker } from 'bullmq'
import Redis from 'ioredis'

// 创建连接
const connection = new Redis('redis://localhost:6379')

// 创建队列
const myQueue = new Queue('my-task', { connection })

// 添加任务（Producer）
await myQueue.add('task-name', { data: 'value' }, {
  attempts: 3,
  backoff: {
    type: 'exponential',
    delay: 2000
  }
})

// 处理任务（Worker）
const worker = new Worker('my-task', async (job) => {
  console.log(`Processing job ${job.id}`)
  // 处理逻辑
}, { connection })

worker.on('completed', (job) => {
  console.log(`Job ${job.id} completed`)
})

worker.on('failed', (job, err) => {
  console.error(`Job ${job?.id} failed:`, err)
})
```

### 任务选项

```typescript
await queue.add('task', payload, {
  attempts: 3,              // 失败重试次数
  backoff: {
    type: 'exponential',    // exponential | fixed
    delay: 2000             // 延迟时间（毫秒）
  },
  delay: 1000,              // 延迟执行时间（毫秒）
  priority: 1,              // 优先级（数字越小优先级越高）
  jobId: 'unique-id',       // 自定义任务ID
  removeOnComplete: true,   // 完成后自动删除
  removeOnFail: false       // 失败后保留
})
```

### 适用场景

- 后台下载文件（你项目的场景）
- 发送邮件/短信
- 图片/视频处理
- 定时任务（Cron）
- 任何需要异步处理的耗时操作

### 与替代方案对比

| 方案 | 特点 |
|------|------|
| **BullMQ + Redis** | 功能丰富，支持持久化、分布式 |
| **Kafka / RabbitMQ** | 更适合海量消息场景 |
| **内存队列** | 简单但不支持持久化，服务重启丢失 |
