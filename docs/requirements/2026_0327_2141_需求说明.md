# 文件离线下载服务 - 需求说明

## 一、系统概述

用户通过网页提交远程文件 URL，由服务器代理下载，生成独立下载链接供用户高速下载。

## 二、技术选型

### 2.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **框架** | Nuxt 4 | 全栈框架，前端同构渲染 |
| **UI 组件** | Element Plus | Vue 3 组件库，功能完善 |
| **数据库** | SQLite | 轻量级，无需额外部署 |
| **文件存储** | 本地磁盘 + Nginx | 高性能文件服务 |
| **任务队列** | Redis + Bull | 高性能队列，支持持久化 |

### 2.3 任务队列流程

```
1. 用户提交下载任务 → API 接收
2. 创建任务记录（SQLite）→ 状态：pending
3. 任务写入 Redis 队列（Bull）
4. Worker 从队列取出任务 → 状态：downloading
5. 实时更新进度到数据库（已下载字节、网速）
6. 下载完成 → 状态：completed
7. 生成下载链接（通过 Nginx 访问）
```

**Bull 特性**：
- 任务持久化到 Redis，重启不丢
- 自动重试机制（失败可配置重试次数）
- 进度跟踪（适合下载进度监控）
- 支持延迟任务、优先级队列

## 三、功能需求

### 3.1 在线提交下载任务

- 用户在网页输入文件 URL（如 GitHub release 文件地址）
- 支持批量 URL 提交
- 自动识别文件名、文件大小（若可获取）

### 3.2 异步下载与状态监控

- 服务器后台异步下载，不阻塞用户请求
- **下载状态**：等待中、下载中、已完成、下载失败
- **实时进度**：已下载字节数 / 总字节数、百分比
- **网速监控**：当前下载速度 (KB/s, MB/s)
- 断点续传支持（大文件）
- 下载失败自动重试（可配置重试次数）

### 3.3 下载链接生成

- 下载完成后生成独立访问链接
- 通过 Nginx 直接服务本地文件，提升下载速度
- 链接可设置有效期（可选）
- 支持大文件分片下载 (Range 请求)

## 四、数据模型

| 字段 | 说明 |
|------|------|
| task_id | 任务唯一ID |
| source_url | 原始下载地址 |
| filename | 保存的文件名 |
| filesize | 文件大小（字节） |
| downloaded | 已下载字节 |
| status | 状态：pending/downloading/completed/failed |
| speed | 当前网速 (bytes/s) |
| created_at | 创建时间 |
| completed_at | 完成时间 |

## 五、非功能需求

- 支持高并发下载任务
- 下载目录可配置（支持多盘存储）
- 下载超时时间可配置
- 任务完成后可配置回调通知
